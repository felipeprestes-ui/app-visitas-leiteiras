# Objetivo
Corrigir o erro CRITICO identificado no APK Android:
Invariant Violation: "main" has not been registered.

# Causa Raiz
O package.json tem "main": "App.js", mas em builds de release Android com Expo SDK 51, o app PRECISA ser registrado explicitamente com registerRootComponent(App).

O erro ocorre porque:
1. O App.js so tem export default function App()
2. O Expo runtime em release build espera que o componente raiz seja registrado via registerRootComponent
3. Sem isso, o app crasha imediatamente com "main has not been registered"

# Correcao Necessaria

## Opcao A (recomendada): Criar arquivo index.js
Criar standalone/index.js com:
```javascript
import { registerRootComponent } from 'expo';
import App from './App';

registerRootComponent(App);
```

E atualizar standalone/package.json:
```json
{
  "main": "index.js",
  ...
}
```

## Opcao B: Modificar App.js
Adicionar no final do standalone/App.js:
```javascript
import { registerRootComponent } from 'expo';
registerRootComponent(App);
```
